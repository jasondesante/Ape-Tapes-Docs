# Epic Future Stuff

A lot has been made, but there's so much left to do. Here are some ideas to get you thinking.  Pay attention to how everything fits together. Each part can feed off of each other to build something greater than the sum of it's parts.

## Players

You can make your own player for Serverless Playlists.  It's as simple as one prompt. Just read the JSON of the playlist format and you'll see where all the data is.  It's already possible to create any type of player that easily accept playlists.  The future is now with infinite players of Serverless Playlists.

The [Infinite player](infinite-players.md) singularity happened in 2024 and keeps getting better!

Imagine a music player in any web3 game.  It should be able to support Serverless Playlists. Or any web3 experience in general. Serverless Playlists can add to any web3 experience, and is easy to implement.

## Discovery Pages

Discovery pages offer fresh interpretations of existing data. They prioritize creativity and innovative perspectives to unveil quality and provide a sense of novelty. The playlist and audio tags help with creating curation sites that offer different interpretations of how to sort the same data.&#x20;

The more data that gets uploaded with these tags, the more discovery pages are needed to rank and surface the growing library.

With shared tags like "genre" and "audio-tag", this is a simple but necessary amount of shared data that curation can be built upon.

## Curation Pages

### Unsorted Files

You can create your own curation pages to help others sort through the mess of files in web3. Follow the rules to convert a plain file into NFT metadata format and you'll be able to add anything into a Serverless Playlist.

Creating a page to curate the raw files and tokens in web3 is not a pretty task, most of the content won't be high quality by nature of it being a firehose of all the stuff.  Curation is the most important thing at this level.  How the raw feeds are presented is also important to enable efficient and accurate curation.&#x20;

Build a playlist creator to further increase the amount of ways playlists can be added into the system.  Every part is open to be enhanced because they're all just ideas trying to be the best version of themselves.

Interesting interpretations of presenting the raw feeds of tokens and files could help curation, but the more tags that are on the files to begin with, the more creative the curation pages could be which leads into the next section.

### Sorted Files

Once something has tags, it still isn't fully curated.  You can narrow things down and add more tags and playlists to curate the library in any direction you want to. Adding curation features to your discovery page is a great combo, because once you discover something you might want to curate it to your tastes. &#x20;

Experimentation on this level could enable better ways of crowd sourcing the curation. Think ranking systems like how Newgrounds did it, or Tinder left/right style ranking, and other stuff like that.  This applies to both unsorted and sorted curation pages.

## Mint Pages

You can create your own pages to mint copies, handle promotions, and staking.  Add more features that all contracts in The Wizard's Library can use.  Or fork the Contract Wizard factory if you want to modify the smart contract. &#x20;

Anything on chain that works for one contract in The Wizard's Library will work for all the others, which is why stuff like the specific promotions are possible, because of that shared interface. Anything built upon The Wizard's Library that's on chain will be compatible with all contracts which means everyone benefits from each other's on chain development. Mint pages turn into fun pages.

There could be a leaderboard style page that shows which tokens have accumulated the most referral points.  There could be pages that involve the referral points and use them for different purposes.&#x20;

It would be cool to have a mint provenance type page that shows a minting family tree of each token in the collection so you can see where a generation 3 or 4 copy came from, you can see it's minting history. That would be a cool page to build if anyone's collections get an interesting minting family tree.

There could be promo pages that have to do with a specific contract, rather than all the contracts in The Wizard's Library like the current Promo Page does. This would let each artist create a promo page that matches their specific vibe.&#x20;

More singular and stylized pages for minting copies, the public minting store, and promo pages could all benefit greatly from vibes.  If the page is dedicated to one specific contract and doesn't have the ui related to switching contracts and the ui related to The Contract Wizard, those more singular pages could feel a lot more cool and vibey to visit.  Those are the types of pages you want direct links to go to because they will feel more like you're going to a special page, if the ui and stuff is customized just for that page.
