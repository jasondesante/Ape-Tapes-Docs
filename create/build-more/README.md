# Build More

## Players

You can make your own player for Serverless Playlists.  It's as simple as reading the json of the playlist format and you'll see where all the data is.  It's already possible to create so many different types of players that easily accept playlists and work flawlessly with one prompt.  The future is now with infinite players of Serverless Playlists.

[Infinite players](infinite-players.md) are already real and have been fun since 2024 when Serverless Playlists were made!

## Curation Pages

You can create your own curation pages to help others sort through all the files in web3.  Using the rules laid out regarding playlist tags and audio upload tags, those tags could help with creating decentralized curation sites that offer different interpretations of how to sort the same data. With a shared amount of tags like genre and audio tag, this will be a simple but necessary amount of shared data that curation can be built upon.

Build a playlist creator to further increase the amount of ways playlists can be added into the system, instead of a playlist viewer.  Every part is open to be enhanced because they're all just ideas and aspiring to be the best version of that idea.

## Mint Pages

You can create your own pages to mint copies, promotions, staking.  Add more features that all contracts in The Wizard's Library can use.  Or fork the Contract Wizard factory if you want to modify the smart contract. &#x20;

Anything on chain that works for one contract in The Wizard's Library will work for all the others, which is why stuff like the specific promotions are possible, because of that shared interface. Anything built upon The Wizard's Library that's on chain will be compatible with all contracts which means everyone benefits from each other's on chain development. Mint pages turn into fun pages.
