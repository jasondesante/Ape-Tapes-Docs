# Upgrading Plain Audio

## Upgrade Your Audio

You can easily upgrade your audio files into metadata with The Metadata Maker. That is one of the main reasons you'd want to create metadata, is to provide the information for the image, audio file, artist, album, etc. This is an important step in making a master token, going from audio to metadata.

To see how [click here](how-to-enhance-any-audio.md).

## Audio Upgrading Explained

To convert audio into metadata, it starts with an uploaded file. Then the url of the audio is put into a JSON file to create the metadata. You can do this easily with [The Metadata Maker](../../).

The [Token Metadata](../../metadata-standards/token-metadata/) page details what a token metadata object looks like. There is where you can read about the different fields of data and why it's structured that way.

In short, the file url gets put into the field `animation_url`, following the same standards as NFT metadata. It's just a normal JSON file. Having the title of the track, artist, duration, image, and other information, it really improves the experience in a huge way.  It doesn't change the music itself, but it's definitely an experience level most people expect, to have an image for the track, to know the duration of the track before playing it, as well as knowing the artist and album names.

For step by step instructions on upgrading audio into metadata, [click here.](how-to-enhance-any-audio.md)

## The Curation Problem

Currently greater than 99% of audio uploaded to Arweave has little to no tags or useful metadata. For most music NFTs, you're lucky to find the artist name.

It's up to us, the listeners, to curate things and add tags when they're missing.  Thankfully this is easily solved with Serverless Playlists and The Metadata Maker. The tools exist to easily enhance the discoverability of your tracks and anything else you enjoy listening to which isn't tagged to your liking.

### Connecting Strands

Without The Metadata Maker, there would be no way to tag the untagged. Without tags, you can't search.  If these other platforms aren't going to do the right thing and coordinate, then we can still wrap their good tracks in metadata to enhance the experience for listeners and future curators. Listeners don't need to suffer because of inconsistent tags and lazy platforms, this is an open system and we can fix things for the next listener.

For files that don't have an artist or genre tag, you can add those tags so the next person can have a better time searching.  By wrapping the audio file into metadata, you can attach all the information and extra files you want to make an enjoyable experience for the listener. This makes it easier for people to search the track, by artist name, genre, or tag.  If the track already has a genre tag, it's more likely to be added to playlists, because it will already be searchable and discoverable.

When you upgrade an audio file into metadata, you're helping increase the discoverability of a track. You're connecting the audio file and image together, adding information about the track, project, artist, and more. This enhances the experience of enjoying the music.&#x20;

### Wrapping Metadata

If you spot some other files that have missing metadata, you can wrap the token/song that has bad metadata into a new metadata file that has the proper tags.  This is how listeners can help their favorite artists, who might have made a mistake along the way, tagging the file, or making the metadata.  The highest quality metadata will be the easiest to search and most accurate.  If the artist has tokens with low quality metadata, and a fan improves it, that is a strong incentive for the artist to point the token to the more improved metadata, and enables decentralized collaboration between the artist and listeners to create the type of metadata the listeners want.

Quality will rise to the top and listeners can help their favorite artists by enhancing their simple audio files and simple nft metadata into more thicc metadata that still references the original source of the files but enhances the experience with more information and tags to search the file.  Enhancing the metadata can only make things better and increase the quality of the listening experience. Good metadata for each track enhances playlists as a whole because no token/track needs to be left behind.
